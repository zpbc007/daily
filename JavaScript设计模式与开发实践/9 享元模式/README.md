#### 享元模式

1. 内部与外部状态
    - 内部状态存储于对象内部
    - 内部状态可以被一些对象共享
    - 内部状态独立于具体的场景，通常不会改变
    - 外部状态取决于具体的场景，并根据场景而变化，外部状态不能被共享

2. 享元模式的目的
    把所有内部状态相同的对象都指定为同一个共享的对象。而外部状态可以从对象身上剥离出来，并存储在外部。
    有多少种内部状态的组合，系统中最多存在多少个共享对象。外部状态存储在外部必要时传入共享对象，组装成为一个完整的对象

3. 享元模式是一种用时间换空间的优化模式

4. 何时使用
    - 程序中有大量相似对象
    - 对象大多数状态可以变为外部状态
